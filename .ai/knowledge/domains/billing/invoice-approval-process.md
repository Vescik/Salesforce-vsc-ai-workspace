# Summary

Invoices generated by KimbleOne billing runs must pass through a two-stage approval process before they can be dispatched to customers. Stage 1 is approved by the Project Manager; Stage 2 by the Finance Controller.

# Details

The approval process is a native Salesforce Approval Process named `Invoice_Approval`. It is triggered when `kmbi__Invoice__c.kmbi__Status__c` changes to `Pending Approval`.

**Stage 1 — Project Manager Approval**
- Approver: `kmbi__Invoice__c.kmbi__Project__r.OwnerId` (Project Owner)
- Rejection action: sets status back to `Draft` and sends email notification
- Approval action: advances to Stage 2

**Stage 2 — Finance Controller Approval**
- Approver: Queue — `Finance_Controllers`
- Rejection action: sets status to `Rejected`, locks record
- Approval action: sets status to `Approved`, triggers dispatch automation

# Known Package Behavior

- The managed package does not have a built-in approval process — the approval process was built as a custom Salesforce configuration on top of KimbleOne.
- If the managed package upgrades change `kmbi__Invoice__c` field picklist values, the approval entry criteria may need updating.
- The `kmbi__Status__c` field is a managed field; its picklist values cannot be freely extended without a package customisation.

# Related Salesforce Objects

- `kmbi__Invoice__c` — main invoice record
- `kmbi__InvoiceLine__c` — line items; read-only during approval

# Related Config Records

- Approval Process: `Invoice_Approval` (Setup > Approval Processes)
- Email Template: `Invoice_Approval_Request`
- Email Template: `Invoice_Rejection_Notification`

# Examples

```
Invoice Status Lifecycle:
Draft → Pending Approval → [Stage 1] → [Stage 2] → Approved → Dispatched
                                      ↓ Reject         ↓ Reject
                                    Draft            Rejected (locked)
```

# Edge Cases

- If the Project Owner is inactive, approval is auto-delegated to the Project Owner's manager.
- Invoices over £50,000 require an additional ad-hoc approval from the CFO (custom flow, not in the standard approval process).
