# QA How-to-Test Documentation

QA how-to-test documents in this folder are generated or maintained as drafts from Work Item artifacts and require human review before use.

Each QA document should include:

- Test cases mapped to acceptance criteria.
- Required user, profile, and Permission Set assumptions.
- Required test data and configuration placeholders.
- Functional, regression, negative, permission/security, and configuration-related tests where applicable.
- Evidence to capture.
- Known limitations and open questions.

Do not invent real test records, users, or configuration values. Do not assume KimbleOne/Kantata managed package internals.
