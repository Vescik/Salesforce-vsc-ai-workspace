# Architecture Documentation

Technical documentation in this folder is generated or maintained as a draft from Work Item artifacts and requires human review before it is treated as authoritative.

Each Work Item document should link or refer to the relevant:

- ADO Work Item.
- Context pack.
- Solution design.
- Design review, if present.
- QA how-to-test document, if applicable.
- Precheck report, if applicable.

Documentation must keep Salesforce metadata changes separate from KimbleOne/Kantata configuration record changes. Do not assume managed package internals or include raw data dumps, secrets, logs, or uncontrolled exports.
