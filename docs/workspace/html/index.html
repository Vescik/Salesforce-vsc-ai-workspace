DevOps Center

The workspace prepares evidence and drafts. DevOps Center remains the official Salesforce metadata promotion mechanism.

Context Data Flow

Knowledge Base Flow

  1. Curated notes live in a separate private Git repository.
  2. `make knowledge-sync` copies reviewed notes into `.ai/knowledge/`.
  3. `make knowledge-index` writes `knowledge-cards.jsonl`.
  4. Context packs include selected cards as supporting evidence.
External KB repo Sync .ai/knowledge knowledge-cards.jsonl Context pack

Config Record Flow

Config records are separate from Salesforce metadata. The workspace can analyze controlled, masked config cards and create review skeletons, but no config apply is implemented.

Config registry Config index Impact report Pack skeleton Manual approval No apply

Azure Wiki Publication Flow

Wiki publication is draft-first: scan, place, preview, review, prepare branch, then push only after explicit human approval.

Docs generated Wiki scanned Placement decision Dry run Human review Draft branch PR / manual merge
